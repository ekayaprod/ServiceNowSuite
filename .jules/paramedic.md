**Learning:** When injecting function strings into dynamically constructed template literals (e.g., `finalCode`), escaping the template interpolation syntax (like `\${findEnvFn}`) causes the generated bookmarklet to reference an undefined variable at runtime, leading to a fatal `ReferenceError`.
**Action:** Unescape the interpolation syntax (use `${findEnvFn}` or `${findEnv.toString()}`) so the literal string of the function is embedded during the initial evaluation, and always wrap the resulting code block in `encodeURIComponent` while keeping the `javascript:` protocol unencoded.
**Learning:** Adding drag-and-drop capability to an injected modal requires attaching specific event listeners (`mousedown` on a header, and `mousemove`/`mouseup` on the document) and strictly removing these listeners when the modal is closed to prevent memory leaks and unintended interference with the host page.
**Action:** Ensure all dynamically added document-level event listeners (like `mousemove` and `mouseup` for dragging) are stored as bound instance methods (e.g., `this.mMove`) and explicitly removed using `removeEventListener` in the teardown/close function of the injected component.
**Learning:** When checking the current URL within ServiceNow bookmarklets to determine if the target page is loaded (e.g., in two-click mode), comparing against a raw `window.location.href` fails because the browser or platform may URL-encode characters like `/` (e.g., `%2Fincident.do`). This causes the bookmarklet to misinterpret the state and incorrectly attempt to reload the page or open a new window instead of executing its actions.
