**Learning:** The creation of complex bookmarklets via nested JS template strings requires very specific encoding strategies. When embedding JS code within a template literal that contains its own template literals (for execution inside the host window later), using single backslashes `\`` and `\${` ensures that the JS engine preserves those characters literally within the final compiled string, allowing the generated code to use template literals natively. Trying to double or triple-escape and regex-replace post-compilation often leads to Syntax Errors.
